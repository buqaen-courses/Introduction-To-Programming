# سیستم عامل: مدیر کامپیوتر

## به زبان ساده

**چه چیزی یاد می‌گیرید:** ویندوز، macOS و لینوکس همه «سیستم عامل» هستند—اما این واقعاً به چه معناست؟ بین فشار دادن دکمه روشن و دیدن دسکتاپ شما چه اتفاقی می‌افتد؟ این مقاله توضیح می‌دهد که سیستم عامل چه می‌کند (اغلب نامرئی)، چگونه سخت‌افزار و برنامه‌های شما را مدیریت می‌کند، و چرا برنامه‌نویسان باید آن را درک کنند.

**نکته برای تازه‌کاران:** هرگز سیستم عامل را مستقیماً «اجرا» نمی‌کنید—آن همیشه آنجاست، در پس‌زمینه در حال اجرا. وقتی یک برنامه باز می‌کنید، سیستم عامل آن را بارگذاری می‌کند. وقتی یک فایل ذخیره می‌کنید، سیستم عامل آن را روی دیسک می‌نویسد. وقتی روی صفحه‌کلید تایپ می‌کنید، سیستم عامل آن کلیدها را به برنامه درست تحویل می‌دهد. این لایه نامرئی بین شما و سخت‌افزار است.

---

## سیستم عامل چیست؟ (تعریف ساده)

یک **سیستم عامل (OS)** نرم‌افزار اصلی است که همه چیز را روی کامپیوتر شما مدیریت می‌کند. آن را به عنوان این‌ها در نظر بگیرید:
- **رهبر** یک ارکستر (هماهنگی همه بخش‌ها)
- **مدیر** یک رستوران (اجرای روان همه چیز)
- **مترجم** بین شما و ماشین (تبدیل کلیک‌های شما به دستورالعمل‌ها)

### تصویر بزرگ

```
┌─────────────────────────────────────────────────────────────────────┐
│                    نحوه تعامل شما با کامپیوترها                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│     شما (کاربر)                                                      │
│       │                                                              │

│       ▼                                                              │
│   ┌──────────────────────────────────────────────────────────────┐ │
│   │                    سیستم عامل                                    │ │
│   │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │ │
│   │  │   رابط      │  │   مدیر      │  │    سیستم    │            │ │

│   │  └─────────────┘  └─────────────┘  └─────────────┘            │ │
│   │  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐            │ │
│   │  │   مدیر      │  │   درایور    │  │   مدیر      │            │ │

│   └──────────────────────────────────────────────────────────────┘ │
│       │                                                              │
│       │ «این برنامه را با این منابع اجرا کن»                       │
│       ▼                                                              │

│   │                     سخت‌افزار                                   │ │
│   │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐          │ │
│   │  │   CPU   │  │   RAM   │  │  دیسک   │  │   GPU   │          │ │

└─────────────────────────────────────────────────────────────────────┘

```

**بدون سیستم عامل:**
- باید دستورالعمل‌های خام را برای انجام هر کاری توسط CPU بنویسید
- باید آدرس‌های حافظه را به صورت دستی مدیریت کنید
- باید دستورات خاص سخت‌افزار را برای هر دستگاه بدانید
- هر برنامه باید برای کامپیوترهای مختلف بازنویسی شود

**با سیستم عامل:**
- فقط روی یک آیکون کلیک کنید تا برنامه اجرا شود
- برنامه‌ها بدون تغییر روی کامپیوترهای مختلف کار می‌کنند
- همه مدیریت‌های سخت‌افزار پیچیده برای شما انجام می‌شود

## سیستم عامل به عنوان مدیر رستوران (تشبیه)

بیایید سیستم عامل را از طریق تشبیه رستوران درک کنیم:

```
┌─────────────────────────────────────────────────────────────────────┐
│              رستوران = کامپیوتر با سیستم عامل                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  🧑‍💼 نقش مدیر (سیستم عامل):                                         │
│  ─────────────────────────────────────────────────                 │
│                                                                      │
│  وقتی یک مشتری (کاربر) می‌رسد:                                       │
│  - آنها را در درب استقبال می‌کند ← رابط کاربر                        │
│  - به آنها یک میز (منابع) اختصاص می‌دهد ← مدیریت حافظه               │
│  - سفارش آنها (درخواست) را می‌گیرد ← مدیریت ورودی                   │
│                                                                      │
│  وقتی کارکنان آشپزخانه (برنامه‌ها) کار می‌کنند:                      │
│  - هماهنگی اینکه چه کسی چه چیزی می‌پزد ← زمان‌بندی پردازه‌ها          │
│  - مدیریت مواد اولیه (داده) ← سیستم فایل                            │
│  - سازماندهی سفارش‌ها ← مدیریت وظایف                                │
│                                                                      │
│  وقتی تجهیزات (سخت‌افزار) استفاده می‌شوند:                            │
│  - اطمینان از کار کردن اجاق‌ها ← درایورهای دستگاه                   │
│  - مدیریت برق ← مدیریت انرژی                                        │
│  - نگهداری فضا ← نگهداری سیستم                                       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

| رستوران | کامپیوتر | عملکرد سیستم عامل |
|---------|---------|-------------------|
| مشتری | کاربر | شخصی که از کامپیوتر استفاده می‌کند |
| گارسون/منو | رابط کاربر | نحوه تعامل شما با سیستم |
| مدیر | سیستم عامل | هماهنگی همه چیز |
| کارکنان آشپزخانه | برنامه‌ها/اپلیکیشن‌ها | کار واقعی را انجام می‌دهند |
| مواد اولیه | داده/فایل‌ها | آنچه برنامه‌ها با آن کار می‌کنند |
| تجهیزات آشپزخانه | سخت‌افزار | اجزای فیزیکی |
| دستور پخت | کد برنامه | دستورالعمل‌ها برای وظایف |

## مسئولیت‌های اصلی سیستم عامل

سیستم عامل چندین شغل کلیدی دارد. بیایید آنها را تجزیه کنیم:

### ۱. مدیریت پردازه (هماهنگ‌کننده وظایف)

**آنچه انجام می‌دهد:** تصمیم می‌گیرد کدام برنامه‌ها، چه زمانی و برای چه مدت اجرا شوند.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    مدیریت پردازه                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  سناریو: شما این برنامه‌ها را باز دارید:                            │
│  🌐 کروم (تماشای ویدیو)                                               │
│  📝 ورد (تایپ سند)                                                    │
│  🎵 اسپاتیفای (پخش موسیقی)                                           │
│                                                                      │
│  اما فقط ۴ هسته CPU دارید...                                          │
│                                                                      │
│  ┌────────────────────────────────────────────────────────────────┐ │
│  │  اشتراک‌گذاری زمان CPU (هزاران بار در ثانیه اتفاق می‌افتد):     │ │
│  │                                                                │ │
│  │  زمان ۱میلی‌ثانیه: کروم (GPU را هم برای ویدیو نیاز دارد)       │ │
│  │  زمان ۲میلی‌ثانیه: ورد (فقط منتظر تایپ شما)                    │ │
│  │  زمان ۳میلی‌ثانیه: اسپاتیفای (CPU کوچک برای دیکد صدا نیاز دارد)│ │
│  │  زمان ۴میلی‌ثانیه: کروم (ادامه دیکد ویدیو)                      │ │
│  │  زمان ۵میلی‌ثانیه: وظایف پس‌زمینه (نگهداری سیستم‌عامل)          │ │
│  │  زمان ۶میلی‌ثانیه: کروم دوباره...                               │ │
│  │                                                                │ │
│  │  این آنقدر سریع اتفاق می‌افتد (میلی‌ثانیه) که آنی به نظر می‌رسد! │ │
│  └────────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  تصمیمات سیستم عامل:                                                │
│  - کروم CPU بیشتری می‌گیرد (ویدیو فشرده است)                         │
│  - اسپاتیفای برش‌های کوچک مداوم می‌گیرد (صدا نمی‌تواند قطع شود)      │
│  - ورد می‌تواند منتظر بماند (تایپ کند است، مقیاس انسان)               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**وظایف کلیدی:**
- 🏃 **ایجاد پردازه**: شروع برنامه‌های جدید
- ⏰ **زمان‌بندی**: دادن زمان CPU به هر برنامه
- 🔪 **خاتمه**: توقف برنامه‌ها وقتی تمام می‌شوند یا خراب می‌شوند
- 🗂️ **هماهنگی**: مدیریت چندین برنامه در حال اجرا به طور همزمان

---

### ۲. مدیریت حافظه (سازماندهنده فضای کار)

**آنچه انجام می‌دهد:** پیگیری اینکه چه داده‌ای کجا در RAM است.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    مدیریت حافظه                                      │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  RAM کامپیوتر شما (مثال ۸ گیگابایت):                                  │
│                                                                      │
│  ┌──────────────────────────────────────────────────────────────┐  │
│  │  آدرس: ۰x۰۰۰۰                                                │  │
│  │  ┌─────────────────────────────────────────────────────────┐ │  │
│  │  │  🖥️ سیستم‌عامل (رزرو شده)                                  │ │  │
│  │  │     ~۲ گیگابایت به طور دائم تخصیص یافته                   │ │  │
│  │  └─────────────────────────────────────────────────────────┘ │  │
│  │  آدرس: ۰x۸۰۰۰۰۰۰۰                                          │  │
│  │  ┌─────────────────────────┐  ┌─────────────────────────┐    │  │
│  │  │   🌐 پردازه کروم        │  │   📝 پردازه ورد         │    │  │
│  │  │   ۴۵۰ مگابایت تخصیص    │  │   ۱۵۰ مگابایت تخصیص    │    │  │
│  │  │   (صفحات وب کش شده)     │  │   (محتوای سند)          │    │  │
│  │  └─────────────────────────┘  └─────────────────────────┘    │  │
│  │  آدرس: ۰xB۰۰۰۰۰۰۰۰                                        │  │
│  │  ┌─────────────────────────┐  ┌─────────────────────────┐    │  │
│  │  │   🎵 پردازه اسپاتیفای   │  │   🔲 فضای آزاد          │    │  │
│  │  │   ۲۰۰ مگابایت تخصیص    │  │   (در دسترس برای        │    │  │
│  │  │   (بافرهای صوتی)        │  │    برنامه‌های جدید)       │    │  │
│  │  └─────────────────────────┘  └─────────────────────────┘    │  │
│  │  آدرس: ۰xFFFFFFFF (پایان ۸ گیگابایت)                       │  │
│  └──────────────────────────────────────────────────────────────┘  │
│                                                                      │
│  مسئولیت‌های سیستم عامل:                                            │
│  ✓ پیگیری اینکه کدام برنامه مالک کدام حافظه است                   │
│  ✓ جلوگیری از دسترسی برنامه‌ها به حافظه یکدیگر                     │
│  ✓ تخصیص حافظه بیشتر وقتی برنامه‌ها درخواست می‌کنند                │
│  ✓ بازپس‌گیری حافظه وقتی برنامه‌ها بسته می‌شوند                     │
│  ✓ انتقال داده به دیسک (حافظه مجازی) وقتی RAM پر است                 │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**وظایف کلیدی:**
- 🎯 **تخصیص**: دادن حافظه به برنامه‌ها
- 🛡️ **محافظت**: جدا نگه داشتن برنامه‌ها (امنیت!)
- ♻️ **بازپس‌گیری**: آزادسازی حافظه وقتی تمام شد
- 💾 **حافظه مجازی**: استفاده از دیسک به عنوان RAM اضافی در صورت نیاز

### ۳. مدیریت سیستم فایل (کتابدار)

**آنچه انجام می‌دهد:** سازماندهی نحوه ذخیره و بازیابی داده روی دیسک.

```
┌─────────────────────────────────────────────────────────────────────┐
│                  مدیریت سیستم فایل                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  وقتی «document.txt» را ذخیره می‌کنید:                                │
│                                                                      │
│  گام ۱: سیستم عامل درخواست را دریافت می‌کند                          │
│          «'document.txt' را در /Users/Alice/Documents/ ذخیره کن»    │
│                                                                      │
│  گام ۲: سیستم عامل مجوزها را بررسی می‌کند                            │
│          «آیا Alice مجوز نوشتن اینجا را دارد؟» → بله ✓             │
│                                                                      │
│  گام ۳: سیستم عامل فضای دیسک را پیدا می‌کند                          │
│          «کدام بخش‌ها آزاد هستند؟» → بخش‌های ۱۰۴۲-۱۰۵۸ در دسترس    │
│                                                                      │
│  گام ۴: سیستم عامل داده را می‌نویسد                                  │
│          ┌──────────┐                                               │
│          │ بلوک ۱  │ ──> بخش ۱۰۴۲ روی دیسک                          │
│          │ بلوک ۲  │ ──> بخش ۱۰۴۳ روی دیسک                          │
│          │ بلوک ۳  │ ──> بخش ۱۰۴۴ روی دیسک                          │
│          └──────────┘                                               │
│                                                                      │
│  گام ۵: سیستم عامل فهرست (جدول فایل) را به‌روزرسانی می‌کند            │
│          «document.txt در بخش‌های ۱۰۴۲-۱۰۵۸، ۱۵ بلوک است»            │
│                                                                      │
│  گام ۶: سیستم عامل موفقیت را تایید می‌کند                            │
│          پیام «فایل ذخیره شد!» به برنامه                            │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**وظایف کلیدی:**
- 📁 **سازماندهی**: مدیریت پوشه‌ها و محل فایل‌ها
- 🔍 **بازیابی**: یافتن سریع فایل‌ها
- 🔐 **مجوزها**: کنترل اینکه چه کسی به چه چیزی دسترسی دارد
- 🛡️ **محافظت**: جلوگیری از خرابی داده

### ۴. مدیریت دستگاه (مترجم سخت‌افزار)

**آنچه انجام می‌دهد:** کار کردن سخت‌افزارهای مختلف با سیستم عامل از طریق **درایورها**.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    مدیریت دستگاه                                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  بدون درایور (هرج‌ومرج غیرممکن):                                     │
│                                                                      │
│  برنامه ──> «هی، صدا را روی HP-LaserJet-Speaker-Model-X7 پخش کن!» │
│               [برنامه باید جزئیات هر دستگاه را بداند!]              │
│                                                                      │
│  با درایور (انتزاع تمیز):                                            │
│                                                                      │
│  برنامه ──> «این صدا را پخش کن» ──> سیستم‌عامل ──> درایور صدا ──> بلندگو │
│  (عمومی)    (درخواست عمومی)   (مسیریابی)   (ترجمه)   (پخش)            │
│                                                                      │
│  درایور بین دستورات عمومی سیستم عامل و                              │
│  دستورالعمل‌های خاص دستگاه ترجمه می‌کند.                              │
│                                                                      │
│  مثال: چاپ یک سند                                                      │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐  │
│  │  ورد     │───> │    سیستم‌عامل    │───> │  درایور  │───> │  کاغذ  │  │
│  │  "چاپ"   │     │ "چاپ    │     │  پرینتر  │     │  چاپ  │  │
│  │          │     │  سند"    │     │ "برو به  │     │         │  │
│  │          │     │          │     │  x,y؛     │     │         │  │
│  │          │     │          │     │  پاشیدن جوهر│    │         │  │
│  └──────────┘     └──────────┘     └──────────┘     └──────────┘  │
│                                                                      │
│  انواع دستگاه‌های رایج:                                               │
│  🖨️ پرینترها      🎧 دستگاه‌های صوتی    📷 دوربین‌ها                     │
│  ⌨️ صفحه‌کلیدها      🖱️ ماوس/پدهای لمسی   📡 کارت‌های شبکه               │
│  🖥️ نمایشگرها       💾 درایوهای USB       🎮 کنترلرهای بازی             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**وظایف کلیدی:**
- 🔄 **ارتباط**: صحبت با سخت‌افزار
- 🌐 **انتزاع**: پنهان کردن پیچیدگی سخت‌افزار
- 🔌 ** plug-and-play**: تشخیص خودکار دستگاه‌های جدید
- ⚡ **مدیریت انرژی**: کنترل حالت‌های sleep/wake

### ۵. رابط کاربر (دروازه شما)

## چرا سیستم‌عامل‌ها برای برنامه‌نویسان مهم هستند

به عنوان کسی که در حال یادگیری برنامه‌نویسی است، درک سیستم‌عامل حیاتی است:

### ۱. مدیریت منابع

درک سیستم‌عامل به شما کمک می‌کند کد کارآمد بنویسید:

```python
# مثال: پایتون به طور خودکار تفاوتهای سیستم‌عامل را مدیریت می‌کند
# اما دانستن سیستم‌عامل به شما کمک می‌کند عملکرد را درک کنید

import os
import psutil  # (pip install psutil)

# بررسی اینکه برنامه شما چقدر حافظه استفاده می‌کند
process = psutil.Process(os.getpid())
print(f"حافظه استفاده شده: {process.memory_info().rss / 1024 / 1024:.2f} مگابایت")

# این مهم است چون:
# - می‌توانید نشتی‌های حافظه را تشخیص دهید
# - می‌توانید استفاده از منابع را بهینه کنید
# - می‌فهمید چرا برنامه شما کند می‌شود
```

### ۲. آگاهی چندسکویی

سیستم‌عامل‌های مختلف رفتارهای مختلفی دارند:

| جنبه | ویندوز | macOS/لینوکس |
|------|---------|-------------|
| **مسیرهای فایل** | `C:\Users\Name\file.txt` | `/Users/Name/file.txt` |
| **پایان خطوط** | `\r\n` (CRLF) | `\n` (LF) |
| **حساسیت به حروف بزرگ** | حساس به حروف بزرگ نیست | حساس به حروف بزرگ است |
| **جداکننده مسیر** | بک‌اسلش `\` | اسلش `/` |

```python
# پایتون به مدیریت تفاوتها کمک می‌کند:
import os
import platform

# به طور خودکار از جداکننده مسیر درست استفاده کنید
path = os.path.join("Users", "Name", "Documents", "file.txt")
# ویندوز: Users\Name\Documents\file.txt
# macOS/لینوکس: Users/Name/Documents/file.txt

# تشخیص سیستم‌عامل
system = platform.system()
if system == "Windows":
    print("روی ویندوز در حال اجرا")
elif system == "Darwin":
    print("روی macOS در حال اجرا")
elif system == "Linux":
    print("روی لینوکس در حال اجرا")
```

### ۳. یکپارچگی سیستم

برنامه‌های شما به طور مداوم با سیستم‌عامل تعامل خواهند داشت:

```python
import os

# متغیرهای محیطی (توسط سیستم‌عامل تنظیم می‌شوند)
home = os.environ.get('HOME')  # /home/username (لینوکس/مک)
                              # C:\Users\Username (ویندوز)

user = os.environ.get('USERNAME')  # نام کاربر فعلی
path = os.environ.get('PATH')       # جایی که سیستم‌عامل برنامه‌ها را جستجو می‌کند

# عملیات فایل (توسط سیستم‌عامل مدیریت می‌شود)
with open("myfile.txt", "r") as f:
    content = f.read()  # سیستم‌عامل از دیسک می‌خواند
```

## سوالات رایج مبتدی‌ها پاسخ داده شد

### سوال: چرا نمی‌توانم این فایل را حذف کنم؟
**پاسخ:** سیستم‌عامل در حال محافظت از آن است. دلایل ممکن:
- فایل در برنامه دیگری باز است (سیستم‌عامل آن را قفل می‌کند)
- شما مجوز ندارید (امنیت سیستم‌عامل)
- این یک فایل سیستمی است (سیستم‌عامل از آسیب تصادفی جلوگیری می‌کند)
- فایل فقط‌خواندنی است (سیستم‌عامل به ویژگی‌های فایل احترام می‌گذارد)

### سوال: چرا کامپیوتر من کند می‌شود؟
**پاسخ:** معمولاً یکی از این منابع مدیریت شده توسط سیستم‌عامل تحت فشار است:
- **RAM پر**: سیستم‌عامل در حال جابه‌جایی به دیسک است (خیلی کند)
- **CPU شلوغ**: برنامه‌های زیادی در حال رقابت برای زمان هستند
- **دیسک پر**: سیستم‌عامل فضایی برای فایل‌های موقت ندارد
- **پردازه‌های پس‌زمینه زیادی**: سیستم‌عامل در حال مدیریت چیزهای زیادی است

### سوال: تفاوت بین Sleep و Hibernate چیست؟
**پاسخ:**
- **Sleep**: سیستم‌عامل RAM را روشن نگه می‌دارد، بقیه را خاموش می‌کند. بیدار شدن سریع، اما برق مصرف می‌کند.
- **Hibernate**: سیستم‌عامل محتویات RAM را روی دیسک ذخیره می‌کند، به طور کامل خاموش می‌شود. بیدار شدن طولانی‌تر، اما برق مصرف نمی‌کند.

### سوال: چرا باید درایوهای USB را "Eject" کنم؟
**پاسخ:** سیستم‌عامل ممکن است:
- هنوز در حال نوشتن داده باشد (نوشتن‌های به تاخیر افتاده برای عملکرد)
- اطلاعات فایل را کش کند
- در برابر خرابی داده محافظت کند
Eject به سیستم‌عامل می‌گوید همه چیز را تمام کند و حذف ایمن شود.

### سوال: "پردازه‌های پس‌زمینه" چیستند؟
**پاسخ:** برنامه‌های در حال اجرا بدون یک پنجره قابل مشاهده:
- سرویس‌های سیستم (شبکه، امنیت)
- برنامه‌های بررسی به‌روزرسانی
- ابزارهای همگام‌سازی ابری (Dropbox، OneDrive)
- مدیریت‌کننده‌های اعلان
آنها از منابع سیستم‌عامل استفاده می‌کنند اما رابط کاربر ندارند.

سعی کنید قبل از ادامه، این‌ها را با کلمات خودتان پاسخ دهید:

۱. **سه کاری که سیستم‌عامل انجام می‌دهد که ممکن است از آنها غافل شوید را نام ببرید.**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   پاسخ‌های ممکن زیاد: کشیدن پنجره‌ها روی صفحه، مدیریت حرکت ماوس، ذخیره فایل‌ها روی دیسک، اجرای چندین برنامه به طور همزمان، اتصال به Wi-Fi، چاپ اسناد، مدیریت ورودی صفحه‌کلید، مدیریت حافظه، و غیره.
   </details>

۲. **"هسته" چیست و چطور با دسکتاپ یا منوی شروع متفاوت است؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   هسته بخش اصلی سیستم‌عامل است که مستقیماً سخت‌افزار را مدیریت می‌کند (CPU، حافظه، دستگاه‌ها). نامرئی است و با بیشترین امتیازات اجرا می‌شود. دسکتاپ و منوی شروع بخشی از لایه رابط کاربر هستند—آنچه می‌بینید و با آن تعامل دارید. هسته مانند موتور یک ماشین است؛ داشبورد مانند دسکتاپ است—اهداف بسیار متفاوتی دارند.
   </details>

۳. **چرا یک برنامه‌نویس باید بداند کاربرانش چه سیستم‌عاملی دارند؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   سیستم‌عامل‌های مختلف فرمت‌های مسیر فایل مختلفی دارند (بک‌اسلش در مقابل اسلش)، ویژگی‌های سیستمی مختلف در دسترس، مدل‌های امنیتی مختلف، و راه‌های مختلفی برای مدیریت عملیات خاص. برنامه‌ها اغلب برای کار کردن صحیح در همه جا به کد خاص سیستم‌عامل نیاز دارند.
   </details>

۴. **در فرآیند بوت چه اتفاقی می‌افتد، به زبان ساده؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   ابتدا، سخت‌افزار خود را بررسی می‌کند (POST). سپس یک برنامه کوچک (بوت‌لودر) هسته سیستم‌عامل را بارگذاری می‌کند. هسته درایورها و توابع اصلی را مقداردهی اولیه می‌کند. سپس سرویس‌های سیستم شروع می‌شوند. در نهایت، رابط کاربر بارگذاری می‌شود و دسکتاپ خود را می‌بینید. مثل بیدار شدن است: بررسی بدن، شروع مغز، فعال کردن حواس، سپس تعامل با دنیا.
   </details>

۵. **چرا به درایورهای دستگاه نیاز داریم؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   هر دستگاه سخت‌افزاری (پرینتر، ماوس، کارت گرافیک) متفاوت کار می‌کند و به دستورالعمل‌های خاص نیاز دارد. درایورها دستورات عمومی سیستم‌عامل را به دستورالعمل‌های خاص دستگاه ترجمه می‌کنند. بدون درایور، سیستم‌عامل نمی‌دانست چگونه با سخت‌افزار شما صحبت کند. مثل داشتن یک مترجم است پس می‌توانید با کسی که زبان متفاوتی صحبت می‌کند صحبت کنید.
   </details>

## عملی: سیستم‌عامل خود را کاوش کنید

### فعالیت ۱: اطلاعات سیستم خود را بررسی کنید

**ویندوز:**
۱. `Win + R` را فشار دهید، `msinfo32` را تایپ کنید، Enter را فشار دهید
۲. کاوش کنید: نام سیستم‌عامل، نسخه، پردازنده، RAM

**macOS:**
۱. Apple menu → About This Mac را کلیک کنید
۲. نمای کلی را ببینید، سپس System Report برای جزئیات

**لینوکس:**
```bash
# اطلاعات سیستم‌عامل
cat /etc/os-release

# اطلاعات سخت‌افزار
lscpu        # اطلاعات CPU
free -h      # اطلاعات حافظه
df -h        # فضای دیسک
```

### فعالیت ۲: مشاهده پردازه‌های در حال اجرا

**ویندوز:**
۱. `Ctrl + Shift + Esc` را فشار دهید (Task Manager)
۲. اگر نیاز است روی "More details" کلیک کنید
۳. برگه‌های Processes و Details را کاوش کنید

**macOS:**
۱. Activity Monitor را باز کنید (Applications > Utilities)
۲. برگه‌های مختلف (CPU، Memory، و غیره) را کاوش کنید

**لینوکس:**
```bash
# لیست پردازه‌ها
ps aux

# نمایشگر تعاملی
htop    # (ابتدا نصب کنید: sudo apt install htop)
```

### فعالیت ۳: کاوش سیستم فایل

به این پوشه‌های مهم پیمایش کنید:

| سیستم‌عامل | اسناد کاربر | پوشه سیستم |
|------------|-------------|-----------|
| ویندوز | `C:\Users\[YourName]\Documents` | `C:\Windows` |
| macOS | `/Users/[YourName]/Documents` | `/System` |
| لینوکس | `/home/[username]/Documents` | `/etc` (پیکربندی)، `/bin` (برنامه‌ها) |
