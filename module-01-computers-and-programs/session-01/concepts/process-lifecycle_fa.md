# چرخه حیات پردازه: چگونه برنامه‌ها زنده می‌شوند و می‌میرند

## به زبان ساده

**چه چیزی یاد می‌گیرید:** هر بار که یک برنامه را باز می‌کنید—یک مرورگر، یک بازی، یک ویرایشگر متن—کامپیوتر شما چیزی به نام "پردازه" ایجاد می‌کند. این مقاله توضیح می‌دهد که این به چه معناست، از لحظه‌ای که کلیک می‌کنید تا زمانی که برنامه را می‌بندید چه اتفاقی می‌افتد، و چرا درک این موضوع به شما کمک می‌کند کامپیوتر را بهتر استفاده (و برنامه‌نویسی) کنید.

**نکته برای تازه‌کاران:** شما قبلاً پردازه‌ها را در عمل دیده‌اید. وقتی کامپیوتر شما با تب‌های باز زیاد قفل می‌کند، یا وقتی از Task Manager (ویندوز) یا Activity Monitor (مک) برای بستن یک برنامه گیرکرده استفاده می‌کنید—شما در حال کار با پردازه‌ها هستید.

---

## پردازه چیست؟ (یک تشبیه)

تصور کنید یک دستور پخت کتاب آشپزی دارید (the **برنامه**). وقتی واقعاً مواد اولیه را جمع‌آوری می‌کنید و شروع به پختن می‌کنید، یک غذا در حال پختن دارید (the **پردازه**). دستور پخت می‌تواند بارها استفاده شود، اما هر جلسه پخت یک پردازه جداگانه است.

| مفهوم | تشبیه آشپزخانه | معادل کامپیوتر |
|-------|-----------------|----------------|
| **برنامه** | کتاب آشپزی روی قفسه | فایل Chrome.exe روی دیسک |
| **پردازه** | در حال پختن یک غذا | مرورگر Chrome در حال اجرا |
| **پردازه‌های متعدد** | پختن چندین غذا | چندین تب Chrome |
| **منابع** | قابلمه، ماهیتابه، فضای اجاق | RAM، زمان CPU |

### برنامه در مقابل پردازه: تفاوت کلیدی

- **برنامه**: یک فایل روی دیسک شما حاوی دستورالعمل‌ها (مانند یک دستور پخت)
- **پردازه**: برنامه در حال اجرا واقعی، با استفاده از حافظه و CPU (مانند یک غذا در حال پختن)

> **مثال:** شما یک برنامه Chrome نصب شده دارید، اما می‌توانید داشته باشید:
> - پنجره اصلی Chrome (۱ پردازه)
> - ۱۰ تب مرورگر (احتمالاً ۱۰ پردازه دیگر)
> - افزونه‌های Chrome (پردازه‌های اضافی)

## سفر یک پردازه: یک داستان گام به گام

بیایید دنبال کنیم چه اتفاقی می‌افتد وقتی روی آیکون یک مرورگر وب دوبار کلیک می‌کنید.

### فاز ۱: تولد (ایجاد پردازه)

**آنچه اتفاق می‌افتد:**
۱. شما روی آیکون مرورگر دوبار کلیک می‌کنید
۲. سیستم‌عامل (OS) می‌گوید «خوب، بیا یک پردازه ایجاد کنیم!»
۳. OS یک "شناسنامه تولد" تخصیص می‌دهد (شناسه پردازه - PID)
۴. OS یک فضای کار آماده می‌کند (تخصیص فضای حافظه)
۵. OS برنامه را از دیسک به حافظه بارگذاری می‌کند
۶. OS منابع اولیه را به پردازه می‌دهد (دسته‌های فایل، مجوزها)

**نمودار بصری:**
```
┌─────────────────────────────────────────────────────────────┐
│                    ایجاد پردازه                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  کاربر کلیک می‌کند ───> OS درخواست را ───> OS یک PID ───> برنامه   │
│  روی آیکون Chrome    دریافت می‌کند    یکتا تخصیص می‌دهد  بارگذاری شد │
│                                                              │
│     👆            🖥️ OS            🏷️ PID #4521     💾➡️💻    │
│                                                              │
│     ↓                ↓                ↓              ↓       │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐  │
│  │           فضای حافظه تخصیص یافته                        │  │
│  │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐    │  │
│  │  │  بخش   │  │  بخش   │  │  پشته  │  │  پشته   │    │  │
│  │  │  کد    │  │  داده  │  │ (محلی)│  │ (پویا)  │   │  │
│  │  └─────────┘  └─────────┘  └─────────┘  └─────────┘    │  │
│  └────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

**شناسه پردازه (PID) چیست؟**
هر پردازه یک عدد یکتا به نام PID دریافت می‌کند. مانند یک برچسب نام است که به OS کمک می‌کند همه پردازه‌های در حال اجرا را پیگیری کند.

---

### فاز ۲: زندگی (اجرای پردازه)

**آنچه اتفاق می‌افتد:**
۱. پردازه شروع به اجرای دستورالعمل‌ها یکی یکی می‌کند
۲. CPU برش‌های کوچکی از زمان (میلی‌ثانیه) به پردازه می‌دهد
۳. پردازه وظایف خود را انجام می‌دهد (بارگذاری صفحات وب، رندر UI)
۴. پردازه به کلیک‌ها و ورودی‌های شما پاسخ می‌دهد

**رقص اشتراک‌گذاری زمان CPU:**
```
زمان →  ────────────────────────────────────────────────────────>

CPU    │    🔄    │    🎨    │    🔄    │    📝    │    🔄    │
به     │ Chrome   │  Paint   │ Chrome   │ Notepad  │ Chrome   │
پردازه │   تب ۱   │          │   تب ۲   │          │   تب ۱   │
       │          │          │          │          │          │
       └──────────┴──────────┴──────────┴──────────┴──────────┘

هر پردازه یک برش کوچک از زمان CPU دریافت می‌کند (میلی‌ثانیه)
این جابه‌جایی آنقدر سریع اتفاق می‌افتد که همزمان احساس می‌شود!
```

**آنچه پردازه در حین اجرا انجام می‌دهد:**
- 🌐 صفحات وب را از اینترنت دریافت می‌کند
- 🎨 گرافیک و متن را روی صفحه رندر می‌کند
- 📊 UI را هنگام اسکرول یا کلیک شما به‌روزرسانی می‌کند
- 💾 بوک‌مارک‌ها یا تاریخچه شما را ذخیره می‌کند

---

### فاز ۳: رشد (درخواست‌های منابع)

همانطور که مرورگر اجرا می‌شود، ممکن است منابع بیشتری نیاز داشته باشد:

```
رشد پردازه در طول زمان:

وضعیت اولیه:              پس از باز کردن ۵ تب:         پس از باز کردن ۲۰ تب:
┌──────────────┐            ┌──────────────────┐          ┌──────────────────────┐
│ Chrome       │            │ Chrome           │          │ Chrome               │
│ ├─ ۵۰MB RAM  │   ───>     │ ├─ ۱۵۰MB RAM     │   ───>   │ ├─ ۸۰۰MB RAM         │
│ ├─ ۱ هسته CPU│            │ ├─ ۲ هسته CPU    │          │ ├─ ۴ هسته CPU        │
│ └─ GPU پایه │            │ ├─ شبکه: ۵x       │          │ ├─ شبکه: ۲۰x         │
│              │            │ └─ GPU: ۲x استفاده│          │ └─ GPU: حداکثر استفاده│
└──────────────┘            └──────────────────┘          └──────────────────────┘
```

**انواع منابعی که یک پردازه می‌تواند درخواست کند:**

| منبع | برای چه استفاده می‌شود | مثال |
|------|------------------------|------|
| **RAM بیشتر** | بارگذاری صفحات وب بزرگ | یک تب با یک بازی ویدیویی |
| **زمان CPU** | محاسبات پیچیده | رندر کردن گرافیک ۳D |
| **شبکه** | دانلود داده | پخش یک ویدیو |
| **دسترسی به دیسک** | ذخیره فایل‌ها | دانلود یک پیوست |
| **GPU** | انیمیشن‌های روان | پخش یک ویدیو |

### فاز ۴: ارتباط (ارتباط بین پردازه‌ای)

پردازه‌ها اغلب نیاز به صحبت با یکدیگر دارند:

```
مثال: ارتباط ابزار اسکرین‌شات

┌───────────────┐      "داده صفحه    ┌───────────────┐
│  ابزار        │ ───── را بده" ────> │  مدیر         │
│  اسکرین‌شات   │                      │  دسکتاپ       │
│               │ <──── "این تصویر" ── │               │
└───────┬───────┘                      └───────────────┘
        │
        │ «این تصویر را ذخیره کن»
        ↓
┌───────────────┐
│  سیستم فایل   │
└───────────────┘
```

**چرا پردازه‌ها ارتباط برقرار می‌کنند:**
- 🔄 اشتراک‌گذاری داده بین برنامه‌ها (کپی-پیست)
- 📢 ارسال اعلانات (تقویم → مرکز اعلانات)
- 🖼️ درخواست خدمات (مرورگر از OS برای انتخاب‌گر فایل درخواست می‌کند)
- 🎵 هماهنگی وظایف (برنامه موسیقی → صدای سیستم)

---

### فاز ۵: مرگ (خاتمه پردازه)

همه چیزهای خوب به پایان می‌رسند. یک پردازه می‌تواند به چندین روش پایان یابد:

```
سناریوهای خاتمه پردازه:

┌─────────────────────────────────────────────────────────────────────┐
│                        انواع خاتمه                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  🎉 خروج عادی          ⚠️ خروج خطا          💥 خرابی                │
│  ───────────────         ──────────────         ─────────            │
│  کاربر روی دکمه X        برنامه یک خطا       خرابی غیرمنتظره       │
│  کلیک می‌کند             را تشخیص می‌دهد       (باگ، مشکل حافظه)    │
│  و خارج می‌شود           و خارج می‌شود                               │
│                                                                      │
│  ✓ خاموش شدن تمیز        ! خطا ثبت شد       ✗ پاکسازی اجباری      │
│  ✓ حافظه آزاد شد         حافظه آزاد شد      حافظه آزاد شد        │
│  ✓ منابع آزاد شدند         منابع آزاد شدند    منابع آزاد شدند      │
│                                                                      │
│  🔪 خاتمه اجباری (Task Manager / دستور kill)                          │
│  ─────────────────────────────────────────────────                 │
│  OS می‌گوید "الان متوقف شو!" و پردازه را مجبور به پایان می‌کند       │
│  برای وقتی استفاده می‌شود که پردازه قفل شده یا بد رفتار می‌کند       │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**آنچه وقتی یک پردازه می‌میرد اتفاق می‌افتد:**
۱. پردازه سیگنال "من تمام شدم" می‌فرستد یا OS مشکل را تشخیص می‌دهد
۲. OS همه حافظه استفاده شده توسط پردازه را بازپس می‌گیرد
۳. همه فایل‌ها و اتصالات بسته می‌شوند
۴. شناسه پردازه بازنشسته می‌شود (می‌تواند بعداً دوباره استفاده شود)
۵. کد خروج ثبت می‌شود (۰ = موفقیت، دیگر = خطا)

## وضعیت‌های پردازه: نمودار چرخه حیات

یک پردازه فقط «در حال اجرا» یا «در حال اجرا نیست» نیست. چندین وضعیت را طی می‌کند:

```
┌─────────────────────────────────────────────────────────────────────┐
│                     نمودار وضعیت پردازه                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│     ┌───────┐                                                       │
│     │  NEW  │  ←── تازه ایجاد شده، منتظر پذیرش                       │
│     └───┬───┘                                                       │
│         │ OS پردازه را می‌پذیرد                                      │
│         ▼                                                           │
│     ┌───────┐     CPU در دسترس       ┌─────────┐                   │
│     │ READY │ ────────────────────> │ RUNNING │                    │
│     └───────┘                       └───┬───┘                      │
│         ▲                               │                         │
│         │  برش زمانی CPU تمام می‌شود    │  نیاز به I/O (دیسک،      │
│         │  یا پردازه با اولویت بالاتر   │  شبکه، ورودی)            │
│         │                               ▼                         │
│     ┌───────┐                       ┌─────────┐                   │
│     │ READY │ <──────────────────── │ WAITING │                   │
│     └───────┘    I/O کامل شد         └────┬────┘                   │
│                                            │                        │
│              پردازه تمام شد، کشته شد،       │                        │
│              یا خراب شد                    ▼                        │
│                                       ┌───────────┐                │
│                                       │ TERMINATED│                │
│                                       └───────────┘                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### درک هر وضعیت

| وضعیت | به چه معناست | مثال دنیای واقعی |
|-------|---------------|------------------|
| **New** | پردازه در حال ایجاد است | کلیک روی آیکون Chrome، منتظر باز شدن آن |
| **Ready** | پردازه آماده اجرا، منتظر CPU | Chrome بارگذاری شده، منتظر نوبت |
| **Running** | پردازه در حال اجرای فعال | Chrome در حال رندر یک صفحه وب |
| **Waiting** | پردازه برای ورودی/خروجی متوقف شده | Chrome منتظر دانلود یک صفحه وب |
| **Terminated** | پردازه تمام شده | پنجره Chrome بسته شد، حافظه پاکسازی شد |

**چرا این مهم است:** وقتی یک برنامه "قفل" می‌کند، معمولاً در وضعیت **Waiting** گیر کرده است، منتظر چیزی که هرگز نمی‌رسد (مانند یک پاسخ شبکه یا قفل فایل).

## جدول پردازه چیست؟

سیستم‌عامل یک لیست اصلی از همه پردازه‌ها را نگه می‌دارد:

```
┌─────────────────────────────────────────────────────────────────────┐
│                    جدول پردازه OS (ساده‌شده)                          │
├─────────┬────────────┬──────────────┬──────────────┬────────────────┤
│  PID    │   نام      │    وضعیت     │ حافظه استفاده │   زمان CPU     │
├─────────┼────────────┼──────────────┼──────────────┼────────────────┤
│   4521  │ chrome.exe │   در حال اجرا │   ۱۵۰ MB     │   ۲.۵ ثانیه    │
│   4523  │ chrome.exe │   در حال اجرا │   ۲۰۰ MB     │   ۳.۱ ثانیه    │
│   2100  │ notepad.exe│   در انتظار  │    ۱۵ MB     │   ۰.۸ ثانیه    │
│   1024  │ explorer.exe│  در حال اجرا  │    ۸۵ MB     │   ۵.۲ ثانیه    │
│   3080  │ spotify.exe│   در حال اجرا │   ۱۲۰ MB     │   ۱.۹ ثانیه    │
└─────────┴────────────┴──────────────┴──────────────┴────────────────┘

هر پردازه یک PID یکتا و آمار پیگیری شده دارد!
```

## تشبیه دنیای واقعی: آشپزخانه رستوران

بیایید چرخه حیات پردازه را با یک رستوران شلوغ مقایسه کنیم:

```
┌─────────────────────────────────────────────────────────────────────┐
│              رستوران = کامپیوتر شما                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  🏪 رستوران          =   💻 سیستم‌عامل                               │
│  ─────────────────────────────────────────────────                   │
│  مدیر که همه چیز را هماهنگ می‌کند                                     │
│                                                                      │
│  📋 کتاب دستور پخت   =   💾 فایل برنامه (روی دیسک)                    │
│  ─────────────────────────────────────────────────                   │
│  دستورالعمل‌ها برای تهیه یک غذا                                      │
│  (فقط آنجا نشسته، هنوز کاری انجام نمی‌دهد)                          │
│                                                                      │
│  👨‍🍳 پختن یک غذا      =   ⚙️ پردازه (برنامه در حال اجرا)              │
│  ─────────────────────────────────────────────────                   │
│  کار فعال تهیه غذا                                                   │
│                                                                      │
│  🔥 ایستگاه‌های آشپزخانه =   🧠 هسته‌های CPU                          │
│  ─────────────────────────────────────────────────                   │
│  جایی که پختن واقعی (اجرای) اتفاق می‌افتد                           │
│                                                                      │
│  📦 انبار/یخچال       =   💾 ذخیره‌سازی (هارد دیسک/SSD)                │
│  ─────────────────────────────────────────────────                   │
│  جایی که مواد اولیه برای مدت طولانی نگه داشته می‌شوند                 │
│                                                                      │
│  🗂️ میزهای آماده‌سازی =   💾 RAM (حافظه)                              │
│  ─────────────────────────────────────────────────                   │
│  فضای کاری برای غذاهای در حال پختن                                   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**تشبیه کامل آشپزخانه:**

| فاز پردازه | معادل آشپزخانه | آنچه اتفاق می‌افتد |
|------------|-----------------|-------------------|
| **ایجاد** | مشتری یک غذا سفارش می‌دهد | برچسب سفارش (PID) ایجاد می‌شود، به یک ایستگاه اختصاص داده می‌شود |
| **اجرا** | سرآشپز غذا می‌پزد | دستور پخت دنبال می‌شود، مواد اولیه آماده می‌شوند |
| **منابع** | نیاز به مواد اولیه بیشتر | سرآشپز موارد اضافی را از انبار درخواست می‌کند |
| **ارتباط** | صدا زدن "سفارش آماده است!" | کارکنان آشپزخانه برای هماهنگی صحبت می‌کنند |
| **خاتمه** | غذا سرو شد، ایستگاه تمیز شد | برچسب دور انداخته شد، ایستگاه برای سفارش بعدی آماده شد |

## چرا چرخه حیات پردازه برای مبتدی‌ها مهم است

درک پردازه‌ها به شما کمک می‌کند:

### ۱. اشکال‌زدایی برنامه‌ها
- بدانید چرا برنامه‌ها قفل می‌کنند یا خراب می‌شوند
- بفهمید که یک برنامه "پاسخگو نیست" در وضعیت در انتظار گیر کرده است
- درک کنید که کشتن یک پردازه عادی است و گاهی لازم است

### ۲. مدیریت منابع
- استفاده از حافظه و CPU را درک کنید
- یاد بگیرید چرا بستن برنامه‌ها منابع را آزاد می‌کند
- ببینید چگونه اقدامات شما بر عملکرد سیستم تاثیر می‌گذارد

### ۳. نوشتن کد بهتر
- برنامه‌هایی را طراحی کنید که تمیز شروع و متوقف شوند
- خطاها را با احترام (خروج مناسب) مدیریت کنید
- منابع را وقتی تمام شد آزاد کنید (بستن فایل‌ها، اتصالات)

### ۴. استفاده کارآمد از کامپیوتر
- برنامه‌های استفاده‌نشده را ببندید تا منابع آزاد شوند
- بفهمید چرا Task Manager مفید است
- عیب‌یابی عملکرد کند

## اشتباهات و سوءتفاهمات رایج مبتدی‌ها

| اشتباه | چرا اشتباه است | درک صحیح |
|--------|----------------|----------|
| **"بستن پنجره برنامه را فوراً متوقف می‌کند"** | برخی پردازه‌های پس‌زمینه ادامه می‌یابند | برخی برنامه‌ها حتی وقتی پنجره بسته است پردازه‌های پس‌زمینه اجرا می‌کنند (سینی سیستم را بررسی کنید) |
| **"یک برنامه قفل شده هنوز از CPU استفاده می‌کند"** | برنامه‌های قفل شده معمولاً در حال انتظار هستند | برنامه‌های قفل شده معمولاً در وضعیت "در انتظار" هستند، منتظر چیزی که هرگز اتفاق نمی‌افتد |
| **"پردازه‌های بیشتر = کامپیوتر کندتر"** | پردازه‌های بیکار چیزی را کند نمی‌کنند | فقط پردازه‌های در حال اجرای فعال به طور قابل توجهی CPU مصرف می‌کنند. داشتن بسیاری از پردازه‌های بیکار طبیعی است. |
| **"کشتن یک پردازه خطرناک است"** | معمولاً بی‌خطر است | کشتن یک پردازه مانند کشیدن دوشاخه یک وسیله برقی است—آزاردهنده برای آن کار، اما عموماً بی‌خطر |
| **"برنامه‌ها می‌توانند به حافظه یکدیگر دسترسی داشته باشند"** | OS این را جلوگیری می‌کند | سیستم‌عامل‌ها حافظه پردازه را برای امنیت جدا نگه می‌دارند |

## عملی: مشاهده پردازه‌ها توسط خودتان

### ویندوز: استفاده از Task Manager
۱. `Ctrl + Shift + Esc` را فشار دهید تا Task Manager باز شود
۲. اگر نمای ساده را دیدید روی "More details" کلیک کنید
۳. روی برگه "Processes" کلیک کنید
۴. می‌توانید ببینید:
   - همه برنامه‌های در حال اجرا
   - پردازه‌های پس‌زمینه
   - استفاده CPU، حافظه، دیسک، شبکه برای هر پردازه
   - شناسه‌های پردازه (در برگه "Details" فعال کنید)

### macOS: استفاده از Activity Monitor
۱. Activity Monitor را باز کنید (از Applications > Utilities)
۲. همه پردازه‌های در حال اجرا را ببینید
۳. برگه‌های CPU، Memory، Energy، Disk، Network را ببینید
۴. برنامه‌های قفل شده را از اینجا اجباری خاتمه دهید

### لینوکس: استفاده از Terminal
```bash
# دیدن همه پردازه‌ها
ps aux

# نمایشگر تعاملی پردازه (نصب با: sudo apt install htop)
htop

# یافتن یک پردازه خاص
ps aux | grep chrome

# کشتن یک پردازه با PID
kill 4521
```

## نکات کلیدی (در یک نگاه)

```
┌─────────────────────────────────────────────────────────────────────┐
│                     خلاصه چرخه حیات پردازه                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ۱. 🔵 NEW       → پردازه ایجاد می‌شود، یک PID دریافت می‌کند         │
│  ۲. 🟡 READY     → منتظر زمان CPU                                    │
│  ۳. 🟢 RUNNING   → در حال اجرای واقعی دستورالعمل‌ها                  │
│  ۴. 🟠 WAITING   → متوقف برای I/O (ورودی/خروجی)                      │
│  ۵. 🔴 TERMINATED→ پردازه پایان می‌یابد، منابع آزاد می‌شوند         │
│                                                                      │
│  📊 برنامه = کتاب دستور پخت (ایستا)                                  │
│  ⚙️ پردازه = جلسه پختن (فعال)                                        │
│  🏷️ PID     = برچسب نام یکتا برای هر پردازه                          │
│  🖥️ OS      = مدیر که همه چیز را هماهنگ می‌کند                        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

۱. **یک پردازه یک برنامه در حال اجرا است** با یک هویت یکتا (PID) و چرخه حیات
۲. **سیستم‌عامل مدیریت می‌کند** ایجاد، زمان‌بندی و خاتمه پردازه
۳. **پردازه‌ها وضعیت‌هایی دارند** که تغییر می‌کنند: New → Ready → Running → Waiting → Terminated
۴. **مدیریت منابع** حیاتی است—پردازه‌ها حافظه، CPU، فایل‌ها را درخواست و آزاد می‌کنند
۵. **خاتمه تمیز** از مشکلات سیستم جلوگیری می‌کند و منابع را برای برنامه‌های دیگر آزاد می‌کند
۶. **شما می‌توانید پردازه‌ها را مشاهده کنید** با استفاده از Task Manager (ویندوز) یا Activity Monitor (مک)

## بررسی سریع (درک خود را بسنجید)

سعی کنید قبل از ادامه، این‌ها را با کلمات خودتان پاسخ دهید:

۱. **تفاوت بین یک "برنامه" (فایل روی دیسک شما) و یک "پردازه" (نمونه در حال اجرا) چیست؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   یک برنامه یک فایل ایستا حاوی دستورالعمل‌ها است (مانند یک کتاب آشپزی). یک پردازه آن برنامه در حال اجرای فعال است، با استفاده از حافظه و زمان CPU (مانند واقعاً پختن یک غذا). یک فایل برنامه می‌تواند بسیاری از پردازه‌ها را ایجاد کند.
   </details>

۲. **چرا بستن برنامه‌های استفاده‌نشده ممکن است کامپیوتر شما را سریع‌تر کند؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   هر پردازه در حال اجرا از RAM و زمان CPU استفاده می‌کند. بستن برنامه‌ها این منابع را برای برنامه‌هایی که واقعاً استفاده می‌کنید آزاد می‌کند. مثل تمیز کردن میز کارتان برای ایجاد فضا برای پروژه فعلی‌تان است.
   </details>

۳. **وقتی یک برنامه "قفل" می‌کند، احتمالاً در کدام وضعیت پردازه گیر کرده است؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   وضعیت "Waiting". برنامه احتمالاً منتظر چیزی است که هرگز نمی‌رسد—مانند یک پاسخ شبکه، یک قفل فایل، یا ورودی کاربر. در حال اجرا نیست، اما خاتمه هم نیافته است.
   </details>

۴. **وقتی پردازه پایان می‌یابد چه اتفاقی برای حافظه پردازه می‌افتد؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   سیستم‌عامل به طور خودکار همه حافظه استفاده شده توسط پردازه را بازپس می‌گیرد (آزاد می‌کند). این حافظه برای استفاده توسط پردازه‌های دیگر در دسترس می‌شود.
   </details>

۵. **آیا دو پردازه می‌توانند یک PID یکسان داشته باشند؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   خیر، هر پردازه یک PID یکتا دارد در حالی که در حال اجراست. وقتی یک پردازه خاتمه می‌یابد، PID آن می‌تواند برای یک پردازه جدید بعداً دوباره استفاده شود، اما هرگز در همان زمان.
   </details>

## تمرین عملی

**این را امتحان کنید:**

۱. مرورگر وب خود را باز کنید
۲. Task Manager (Ctrl+Shift+Esc در ویندوز) یا Activity Monitor (مک) را باز کنید
۳. مرورگر خود را در لیست پردازه پیدا کنید
۴. PID و استفاده حافظه آن را یادداشت کنید
۵. ۵ تب دیگر در مرورگر خود باز کنید
۶. تماشای تغییر استفاده حافظه در Task Manager
۷. آن تب‌ها را ببندید و دوباره استفاده حافظه را تماشا کنید

**سوالات برای فکر کردن:**
- آیا تعداد پردازه‌ها وقتی تب‌های بیشتری باز کردید تغییر کرد؟
- آیا استفاده حافظه افزایش یافت؟ چقدر؟
- آیا حافظه کاملاً به سطح اصلی برگشت وقتی تب‌ها را بستید؟

## مطالعه و کاوش بیشتر

- **خودتان امتحان کنید:** با Task Manager/Activity Monitor آزمایش کنید تا ببینید چند پردازه روی سیستم شما اجرا می‌شود
- **بیشتر بیاموزید:** الگوریتم‌های زمان‌بندی پردازه (Round Robin، زمان‌بندی اولویت) را مطالعه کنید
- **کاوش کنید:** چندنخی (پردازه‌ها می‌توانند چندین رشته اجرا داشته باشند)
- **پیشرفته:** درباره همگام‌سازی پردازه و پیشگیری از بن‌بست بیاموزید
- **مقاله بعدی:** به [سیستم‌عامل](operating-system.md) ادامه دهید تا یاد بگیرید سیستم‌عامل چگونه همه این پردازه‌ها را مدیریت می‌کند
