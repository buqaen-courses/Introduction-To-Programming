# فرمت باینری: زبان کامپیوترها

## به زبان ساده

**چه چیزی یاد می‌گیرید:** احتمالاً شنیده‌اید که کامپیوترها از «۰ و ۱» استفاده می‌کنند—اما چرا؟ و چگونه دو رقم می‌توانند همه چیز—متن، تصاویر، موسیقی، ویدیوها، بازی‌های مورد علاقه شما—را نمایش دهند؟ این مقاله دلیل الکتریکی ساده پشت باینری، نحوه مقیاس‌پذیری آن برای نمایش هر داده‌ای، و چرا نیازی به «فکر کردن به باینری» برای برنامه‌نویسی ندارید (زبان‌ها آن را برای شما انجام می‌دهند) را توضیح می‌دهد.

**نکته برای تازه‌کاران:** نیازی به تبدیل اعداد به باینری برای نوشتن کد ندارید. زبان‌های برنامه‌نویسی این کار را به طور خودکار انجام می‌دهند. درک باینری به شما کمک می‌کند *چرا* کامپیوترها اینگونه کار می‌کنند—و چرا اندازه فایل‌ها با بایت، کیلوبایت و گیگابایت اندازه‌گیری می‌شود.

## چرا باینری؟ واقعیت الکتریکی

کامپیوترها ماشین‌های الکتریکی هستند. در پایین‌ترین سطح، آن‌ها فقط دو حالت را درک می‌کنند: **روشن** (جریان برق) و **خاموش** (بدون جریان برق). «نیمه روشن» یا «متوسط» وجود ندارد—فقط دو حالت واضح و متمایز.

```
┌─────────────────────────────────────────────────────────────────────┐
│              چرا کامپیوترها از باینری استفاده می‌کنند                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │

│  ═══════════════════════════════════════════════════════════        │
│                                                                      │
│  برق در یک سیم فقط دو حالت قابل اعتماد دارد:                        │
│                                                                      │
│  ⚡ روشن (ولتاژ موجود)         🔌 خاموش (بدون ولتاژ)                │
│        │                              │                              │
│        │ 5 ولت                      │ 0 ولت                      │
│        │ █████████████               │                              │
│        │███████████████               │░░░░░░░░░░░░░░░░░░░░░░░░░░░  │

│  چرا حالت‌های بیشتر نه؟                                             │

│     • تشخیص ۰v، ۰.۵v، ۱v، ۱.۵v... سخت است                           │
│     • ولتاژ به دلیل گرما، تداخل نوسان می‌کند                        │
│     • خطاها رایج خواهند بود                                         │
│                                                                      │
│  ✅ ۲ حالت (۰ و ۱):                                                 │
│     • تشخیص آسان: «ولتاژ هست یا نه؟»                                │
│     • حاشیه خطای گسترده                                             │
│     • قابل اعتماد و سریع                                            │
│     • مدارهای ساده                                                  │

│  تشبیه: تشخیص روشن یا خاموش بودن یک چراغ آسان‌تر است                │

│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## درک اعداد باینری

### باینری در مقابل دهدهی

ما انسان‌ها از **دهدهی** (پایه ۱۰) استفاده می‌کنیم چون ۱۰ انگشت داریم. کامپیوترها از **باینری** (پایه ۲) استفاده می‌کنند چون ۲ حالت دارند.

```
┌─────────────────────────────────────────────────────────────────────┐
│                    مقایسه سیستم‌های عددی                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  دهدهی (پایه ۱۰) - آنچه انسان‌ها استفاده می‌کنند:                    │
│  ═══════════════════════════════════════════════════════════        │
│                                                                      │
│  ارقام: ۰، ۱، ۲، ۳، ۴، ۵، ۶، ۷، ۸، ۹                               │
│                                                                      │
│  شمارش: ۰ → ۱ → ۲ → ۳ → ۴ → ۵ → ۶ → ۷ → ۸ → ۹                      │
│           ↓ (ارقام تمام شد، مکان جدید اضافه کن)                    │
│           ۱۰ → ۱۱ → ۱۲ → ... → ۹۹                                   │
│           ↓ (دوباره تمام شد، مکان دیگر اضافه کن)                    │
│           ۱۰۰ → ۱۰۱ → ...                                          │
│                                                                      │
│  ارزش مکان‌ها:                                                      │
│  ۱۰۰۰s │ ۱۰۰s │ ۱۰s │ ۱s                                            │
│    ۴      ۲      ۷     ۹    = ۴۰۰۰ + ۲۰۰ + ۷۰ + ۹ = ۴۲۷۹           │
│                                                                      │
│  ─────────────────────────────────────────────────────────────       │
│                                                                      │
│  باینری (پایه ۲) - آنچه کامپیوترها استفاده می‌کنند:                 │
│  ═══════════════════════════════════════════════════════════        │
│                                                                      │
│  ارقام: ۰، ۱ (به نام «بیت» - رقم باینری)                            │
│                                                                      │
│  شمارش: ۰ → ۱                                                       │
│           ↓ (ارقام تمام شد، مکان جدید اضافه کن)                     │
│           ۱۰ → ۱۱                                                   │
│           ↓ (دوباره تمام شد، مکان دیگر اضافه کن)                     │
│           ۱۰۰ → ۱۰۱ → ۱۱۰ → ۱۱۱                                    │
│           ↓                                                        │
│           ۱۰۰۰ → ...                                              │
│                                                                      │
│  ارزش مکان‌ها:                                                      │
│  ۸s │ ۴s │ ۲s │ ۱s                                                  │
│   ۱    ۰    ۱    ۱    = ۸ + ۰ + ۲ + ۱ = ۱۱ (دهدهی)                  │
│                                                                      │
│  هر موقعیت ۲× ارزش موقعیت سمت راست خود را دارد:                     │
│  ۱۲۸  ۶۴   ۳۲   ۱۶    ۸    ۴    ۲    ۱                             │
│   │    │    │    │    │    │    │    │                              │
│   ▼    ▼    ▼    ▼    ▼    ▼    ▼    ▼                              │
│  ┌────┬────┬────┬────┬────┬────┬────┬────┐                         │
│  │۱۲۸s│۶۴s │۳۲s│۱۶s│ ۸s│ ۴s│ ۲s│ ۱s│  ← ارزش مکان‌ها              │
│  └────┴────┴────┴────┴────┴────┴────┴────┘                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## شمارش باینری: الگو را تماشا کنید

بیایید از ۰ تا ۱۰ در هر دو سیستم بشماریم:

```
┌─────────────────────────────────────────────────────────────────────┐
│                    مقایسه شمارش                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  دهدهی │ باینری │ نحوه کار باینری                                   │
│  ───────┼────────┼──────────────────────────────────────────────────│
│    ۰     │   ۰    │ صفر                                              │
│    ۱     │   ۱    │ یک                                               │
│    ۲     │  ۱۰    │ دو: ارقام تمام شد، مکان اضافه (۱ دو + ۰ یک)       │
│    ۳     │  ۱۱    │ دو + یک = سه                                     │
│    ۴     │ ۱۰۰    │ چهار: دوباره تمام شد (۱ چهار + ۰ دو + ۰ یک)      │
│    ۵     │ ۱۰۱    │ چهار + یک = پنج                                  │
│    ۶     │ ۱۱۰    │ چهار + دو = شش                                   │
│    ۷     │ ۱۱۱    │ چهار + دو + یک = هفت                             │
│    ۸     │۱۰۰۰    │ هشت: مکان جدید (۱ هشت + ۰ + ۰ + ۰)                │
│    ۹     │۱۰۰۱    │ هشت + یک = نه                                    │
│   ۱۰     │۱۰۱۰    │ هشت + دو = ده                                    │
│                                                                      │
│  الگو را ببینید:                                                     │
│  • در دهدهی، وقتی به ۹ می‌رسیم مکان اضافه می‌کنیم                   │
│  • در باینری، وقتی به ۱ می‌رسیم مکان اضافه می‌کنیم                  │
│  • اعداد باینری سریع‌تر بلند می‌شوند!                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## بیت، بایت و واحدهای بزرگ‌تر

داده باینری در اندازه‌های استاندارد ارائه می‌شود:

```
┌─────────────────────────────────────────────────────────────────────┐
│                    واحدهای ذخیره‌سازی داده                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  بیت (رقم باینری)                                                    │
│  ═══════════════════════════════════════════════════════════        │
│  • کوچک‌ترین واحد: ۰ یا ۱                                            │
│  • یک سوئیچ، یک سیم، یک قطعه کوچک اطلاعات                            │
│                                                                      │
│  ۱ بیت = ۲ مقدار ممکن: ۰ یا ۱                                       │
│                                                                      │
│  ─────────────────────────────────────────────────────────────       │
│                                                                      │
│  بایت (۸ بیت) - واحد استاندارد                                     │
│  ═══════════════════════════════════════════════════════════        │
│  • ۸ بیت گروه‌بندی شده                                               │
│  • می‌تواند ۲۵۶ مقدار مختلف نمایش دهد (۲^۸)                           │
│  • کافی برای تمام حروف، اعداد و نمادها                               │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                    یک بایت                                     │   │
│  │  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐          │   │
│  │  │ بیت │ بیت │ بیت │ بیت │ بیت │ بیت │ بیت │ بیت │          │   │
│  │  │  ۷  │  ۶  │  ۵  │  ۴  │  ۳  │  ۲  │  ۱  │  ۰  │          │   │
│  │  ├─────┼─────┼─────┼─────┼─────┼─────┼─────┤─────┤          │   │
│  │  │ ۱۲۸ │ ۶۴  │ ۳۲  │ ۱۶  │  ۸  │  ۴  │  ۲  │  ۱  │ = مقدار  │   │
│  │  │  ۰  │  ۱  │  ۰  │  ۰  │  ۰  │  ۰  │  ۰  │  ۱  │ = ۶۵     │   │
│  │  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘          │   │
│  │                    ↑                                          │   │
│  │              این بایت = ۶۵ = حرف 'A' در ASCII               │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  ۱ بایت = ۲۵۶ مقدار ممکن (۰ تا ۲۵۵)                                 │
│                                                                      │
│  ─────────────────────────────────────────────────────────────       │
│                                                                      │
│  واحدهای بزرگ‌تر:                                                    │
│  ═══════════════════════════════════════════════════════════        │
│                                                                      │
│  ۱ کیلوبایت (KB)  = ۱,۰۲۴ بایت       (~۱ هزار)                     │
│  ۱ مگابایت (MB)  = ۱,۰۲۴ KB          (~۱ میلیون)                   │
│  ۱ گیگابایت (GB)  = ۱,۰۲۴ MB          (~۱ میلیارد)                   │
│  ۱ ترابایت (TB)  = ۱,۰۲۴ GB          (~۱ تریلیون)                    │
│  ۱ پتابایت (PB)  = ۱,۰۲۴ TB          (مراکز داده عظیم)              │
│                                                                      │
│  چرا ۱,۰۲۴ و نه ۱,۰۰۰؟                                              │
│  چون ۱,۰۲۴ = ۲^۱۰ (یک عدد گرد در باینری!)                           │
│                                                                      │
│  اندازه‌های دنیای واقعی:                                            │
│  • فایل متنی: چند KB                                               │
│  • عکس: چند MB                                                     │
│  • فیلم: چند GB                                                    │
│  • هارد دیسک: صدها GB تا چند TB                                    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## نحوه ذخیره متن: ASCII و یونیکد

```
┌─────────────────────────────────────────────────────────────────────┐
│                    کدگذاری متن: ASCII                                │

│                                                                      │
│  کدهای ASCII رایج:                                                   │
│  ═══════════════════════════════════════════════════════════        │
│                                                                      │
```

## عملیات باینری (نحوه محاسبه کامپیوترها)

```
┌─────────────────────────────────────────────────────────────────────┐
│                    حساب باینری                                       │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │

│  ═══════════════════════════════════════════════════════════        │
│                                                                      │
│  قوانین:                                                             │
│  ۰ + ۰ = ۰                                                           │
│  ۰ + ۱ = ۱                                                           │
│  ۱ + ۰ = ۱                                                           │
│  ۱ + ۱ = ۱۰  (۰ با رقم نقلی ۱)                                       │
│  ۱ + ۱ + ۱ = ۱۱ (۱ با رقم نقلی ۱)                                    │
│                                                                      │
│  مثال: ۵ + ۳                                                        │
│  ═══════════════════════════════════════════════════════════         │
│                                                                      │
│    دهدهی ۵ = ۰۱۰۱                                                  │
│  + دهدهی ۳ = ۰۰۱۱                                                  │
│  ────────────────                                                    │
│               ۱۰۰۰  = دهدهی ۸                                       │
│                                                                      │
│  گام به گام:                                                         │
│    ۰۱۰۱                                                              │
│  + ۰۰۱۱                                                              │
│  ────────                                                            │
│  موقعیت ۱ (راست‌ترین): ۱ + ۱ = ۱۰ → بنویس ۰، رقم نقلی ۱            │
│  موقعیت ۲: ۰ + ۱ + رقم نقلی ۱ = ۱۰ → بنویس ۰، رقم نقلی ۱             │
│  موقعیت ۳: ۱ + ۰ + رقم نقلی ۱ = ۱۰ → بنویس ۰، رقم نقلی ۱            │
│  موقعیت ۴: ۰ + ۰ + رقم نقلی ۱ = ۱ → بنویس ۱                         │
│                                                                      │
│  نتیجه: ۱۰۰۰ = ۸ ✓                                                  │
│                                                                      │
│  ─────────────────────────────────────────────────────────────       │
│                                                                      │

│  ═══════════════════════════════════════════════════════════        │

│  برای مقایسه و دستکاری بیت‌ها استفاده می‌شوند                        │
│                                                                      │
│  AND (&): هر دو باید ۱ باشند تا ۱ بگیریم                             │
│    ۱ & ۱ = ۱                                                         │
│    ۱ & ۰ = ۰                                                         │
│    ۰ & ۰ = ۰                                                         │
│    استفاده: ماسک کردن بیت‌ها (نگه داشتن فقط بیت‌های خاص)            │
│                                                                      │
│  OR (|): حداقل یکی باید ۱ باشد تا ۱ بگیریم                           │
│    ۱ | ۱ = ۱                                                         │
│    ۱ | ۰ = ۱                                                         │
│    ۰ | ۰ = ۰                                                         │
│    استفاده: ترکیب پرچم‌ها، تنظیم بیت‌ها                              │
│                                                                      │
│  NOT (~): معکوس کردن همه بیت‌ها                                     │
│    ~۱ = ۰                                                            │
│    ~۰ = ۱                                                            │
│    استفاده: معکوس‌سازی، نقیض                                          │
│                                                                      │
│  XOR (^): بیت‌های متفاوت ۱ می‌دهند                                   │
│    ۱ ^ ۱ = ۰                                                         │
│    ۱ ^ ۰ = ۱                                                         │
│    ۰ ^ ۰ = ۰                                                         │
│    استفاده: رمزنگاری، تغییر وضعیت بیت‌ها                              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## بررسی سریع (درک خود را بسنجید)

سعی کنید قبل از ادامه، این‌ها را با کلمات خودتان پاسخ دهید:

۱. **چرا کامپیوترها به راحتی نمی‌توانند از دهدهی (۰-۹) به جای باینری (۰-۱) استفاده کنند؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   کامپیوترها ماشین‌های الکتریکی هستند. تشخیص بین «جریان برق» (۱) و «بدون جریان برق» (۰) آسان است. تشخیص ۱۰ سطح ولتاژ مختلف (۰v، ۱v، ۲v...۹v) به دلیل نویز الکتریکی، گرما و تداخل سخت خواهد بود. دو حالت ساده، قابل اعتماد و سریع است.
   </details>

۲. **۸ بیت (۱ بایت) چند مقدار مختلف می‌تواند نمایش دهد؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   ۲۵۶ مقدار (۰ تا ۲۵۵). با ۸ بیت، هر کدام می‌تواند ۰ یا ۱ باشد، که ۲^۸ = ۲۵۶ ترکیب می‌دهد. به همین دلیل یک بایت می‌تواند هر کاراکتر ASCII (که از مقادیر ۰-۱۲۷ استفاده می‌کند) را ذخیره کند و واحد استاندارد برای کدگذاری کاراکتر است.
   </details>

۳. **وقتی حرف "A" را تایپ می‌کنید، در داخل کامپیوتر چه شکلی به خود می‌گیرد؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   حرف "A" به صورت عدد ۶۵ (در ASCII/یونیکد) ذخیره می‌شود، که در باینری ۰۱۰۰۰۰۰۱ است. این الگوی ۸ بیتی در حافظه ذخیره می‌شود. صفحه‌کلید این کد را وقتی A را فشار می‌دهید ارسال می‌کند، و نمایشگر آن را برای نمایش حرف روی صفحه ترجمه می‌کند.
   </details>

۴. **چرا یک فایل تصویر ۱۹۲۰×۱۰۸۰ خیلی بزرگ‌تر از یک فایل متنی با ۱,۰۰۰ کاراکتر است؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   متن: ۱,۰۰۰ کاراکتر = ۱,۰۰۰ بایت (۱ کیلوبایت). تصویر: ۱۹۲۰ × ۱۰۸۰ پیکسل = حدود ۲ میلیون پیکسل. هر پیکسل به ۳ بایت (رنگ RGB) نیاز دارد، پس ۶ میلیون بایت (۶ مگابایت) فشرده‌نشده. تصاویر حجم داده خام خیلی بیشتری از متن دارند.
   </details>

۵. **نیازی به نوشتن کد باینری ندارید، پس اصلاً چرا باینری یاد بگیریم؟**
   <details>
   <summary>برای پاسخ کلیک کنید</summary>
   درک باینری به شما کمک می‌کند: اندازه فایل‌ها و چرایی پر شدن حافظه را درک کنید، محدودیت‌های حافظه را درک کنید، اهمیت فشرده‌سازی را بدانید، نحوه نمایش داده را درک کنید، انواع خاصی از مشکلات را رفع کنید، و با سایر برنامه‌نویسان بهتر ارتباط برقرار کنید. دانش بنیادی درباره نحوه کار کامپیوترها است.
   </details>

## مطالعه و کاوش بیشتر

- **امتحان کنید:** یک برنامه ساده بنویسید که متن را به باینری تبدیل کند
- **بازی کنید:** بازی‌ها و مبدل‌های باینری آنلاین
- **گام بعدی:** یادگیری یک زبان برنامه‌نویسی را شروع کنید! (پایتون برای مبتدی‌ها عالی است)
