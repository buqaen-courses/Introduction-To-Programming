# سیستم هگزادسیمال: مخفف دودویی

## مقدمه: چرا به هگزادسیمال نیاز داریم؟

تصور کنید دارید یک شماره تلفن خیلی طولانی را می‌نویسید. به جای نوشتن:
```
۰-۰-۰-۱-۰-۱-۰-۱-۰-۱-۱-۰-۰-۱-۱-۰-۰
```

آسان‌تر نیست اگر بنویسید:
```
0x5AC
```

این دقیقاً کاری است که هگزادسیمال ("هگز" برای کوتاه) انجام می‌دهد! راهی فشرده برای نوشتن اعداد دودویی است که انسان‌ها می‌توانند به راحتی بخوانند.

---

## هگزادسیمال چیست؟

### اصول

**هگزادسیمال** یک سیستم اعداد با **۱۶ نماد** است:
- **۰-۹** (مانند دهدهی)
- **A-F** (برای مقادیر ۱۰-۱۵)

| هگز | دهدهی | دودویی |
|-----|-------|--------|
| ۰ | ۰ | ۰۰۰۰ |
| ۱ | ۱ | ۰۰۰۱ |
| ۲ | ۲ | ۰۰۱۰ |
| ۳ | ۳ | ۰۰۱۱ |
| ۴ | ۴ | ۰۱۰۰ |
| ۵ | ۵ | ۰۱۰۱ |
| ۶ | ۶ | ۰۱۱۰ |
| ۷ | ۷ | ۰۱۱۱ |
| ۸ | ۸ | ۱۰۰۰ |
| ۹ | ۹ | ۱۰۰۱ |
| A | ۱۰ | ۱۰۱۰ |
| B | ۱۱ | ۱۰۱۱ |
| C | ۱۲ | ۱۱۰۰ |
| D | ۱۳ | ۱۱۰۱ |
| E | ۱۴ | ۱۱۱۰ |
| F | ۱۵ | ۱۱۱۱ |

### چرا مبنای ۱۶؟

۱۶ = ۲⁴

**این جادو است!** هر ۴ بیت دودویی دقیقاً برابر با ۱ رقم هگزادسیمال است:
```
دودویی:  ۱۱۱۱  ۱۰۱۰  ۱۱۰۰  ۱۱۱۱
         ↓     ↓     ↓     ↓
هگز:      F     A     C     F
```

این تبدیل بین دودویی و هگز را بسیار آسان می‌کند!

## تبدیل دودویی به هگزادسیمال

### قاعده ساده

**بیت‌های دودویی را در مجموعه‌های ۴ تایی از راست گروه‌بندی کنید.**

### مثال ۱: تبدیل ۱۰۱۰۱۱۰۱ به هگزادسیمال

```
مرحله ۱: از راست گروه‌بندی در ۴ تایی
        ۱۰۱۰ ۱۱۰۱

مرحله ۲: تبدیل هر گروه
        ۱۰۱۰ = ۱۰ = A
        ۱۱۰۱ = ۱۳ = D

مرحله ۳: ترکیب
        ۱۰۱۰۱۱۰۱₂ = AD₁₆
```

### مثال ۲: تبدیل ۱۱۱۱۰۰۰۰۱۰۱۰ به هگزادسیمال

```
مرحله ۱: از راست گروه‌بندی در ۴ تایی
        ۱۱۱۱ ۰۰۰۰ ۱۰۱۰
        
مرحله ۲: تبدیل هر گروه
        ۱۱۱۱ = ۱۵ = F
        ۰۰۰۰ = ۰
        ۱۰۱۰ = ۱۰ = A
        
مرحله ۳: ترکیب
        ۱۱۱۱۰۰۰۰۱۰۱۰₂ = F0A₁₆
```

### اگر دودویی به طور مساوی تقسیم نشود چه؟

صفرهای پیشرو اضافه کنید تا گروه‌ها کامل شوند:

**مثال: تبدیل ۱۱۰۱۰۱ به هگزادسیمال**
```
اصلی:      ۱۱۰۱۰۱
۲ صفر اضافه کنید: ۰۰۱۱ ۰۱۰۱

تبدیل:
    ۰۰۱۱ = ۳
    ۰۱۰۱ = ۵

نتیجه: ۳۵۱₆
```

## تبدیل هگزادسیمال به دودویی

### فرآیند معکوس

**هر رقم هگزادسیمال را به ۴ بیت دودویی تبدیل کنید.**

### مثال ۱: تبدیل ۳F به دودویی

```
۳ = ۰۰۱۱
F = ۱۱۱۱

۳F₁₆ = ۰۰۱۱۱۱۱۱₂
```

### مثال ۲: تبدیل A2B به دودویی

```
A = ۱۰۱۰
۲ = ۰۰۱۰
B = ۱۰۱۱

A2B₁₆ = ۱۰۱۰۰۰۱۰۱۰۱۱₂
```

### مثال ۳: تبدیل FF00 به دودویی

```
F = ۱۱۱۱
F = ۱۱۱۱
۰ = ۰۰۰۰
۰ = ۰۰۰۰

FF00₁₆ = ۱۱۱۱۱۱۱۱۰۰۰۰۰۰۰۰₂
```

## تبدیل هگزادسیمال به دهدهی

### روش ۱: از طریق دودویی

هگزادسیمال → دودویی → دهدهی:

**مثال: تبدیل 2A به دهدهی**
```
2A₁₆ = ۰۰۱۰۱۰۱۰₂
       ↓
     ۳۲ + ۸ + ۲ = ۴۲₁₀
```

### روش ۲: ارزش مکانی مستقیم

ارزش‌های مکانی هگزادسیمال توان‌های ۱۶ هستند:
```
موقعیت:   ۲      ۱      ۰
مقدار:   ۲۵۶     ۱۶      ۱
```

**مثال: تبدیل 2A به دهدهی**
```
2A₁₆ = ۲ × ۱۶¹ + ۱۰ × ۱۶⁰
     = ۲ × ۱۶ + ۱۰ × ۱
     = ۳۲ + ۱۰
     = ۴۲₁₀
```

**مثال: تبدیل 1F4 به دهدهی**
```
1F4₁₆ = ۱ × ۱۶² + ۱۵ × ۱۶¹ + ۴ × ۱۶⁰
      = ۱ × ۲۵۶ + ۱۵ × ۱۶ + ۴ × ۱
      = ۲۵۶ + ۲۴۰ + ۴
      = ۵۰۰₁₀
```

## تبدیل دهدهی به هگزادسیمال

### روش ۱: از طریق دودویی

دهدهی → دودویی → هگزادسیمال:

**مثال: تبدیل ۴۵ به هگزادسیمال**
```
۴۵₁₀ = ۰۰۱۰۱۱۰۱₂
         ↓
       ۰۰۱۰ ۱۱۰۱
         ↓
         ۲    D
         ↓
       2D₁₆
```

### روش ۲: تقسیم بر ۱۶

**مثال: تبدیل ۵۰۰ به هگزادسیمال**
```
۵۰۰ ÷ ۱۶ = ۳۱ باقی‌مانده ۴
۳۱  ÷ ۱۶ = ۱  باقی‌مانده ۱۵ (F)
۱   ÷ ۱۶ = ۰  باقی‌مانده ۱

خواندن باقی‌مانده‌ها از پایین به بالا: 1F4₁₆
```

## جایی که هگزادسیمال را در زندگی واقعی می‌بینید

### ۱. رنگ‌های وب

طراحان وب از هگزادسیمال برای رنگ‌ها استفاده می‌کنند:

```css
/* قرمز، سبز، آبی (هر کدام ۰-۲۵۵) */
#FF0000 = قرمز خالص   (۲۵۵, ۰, ۰)
#00FF00 = سبز خالص  (۰, ۲۵۵, ۰)
#0000FF = آبی خالص   (۰, ۰, ۲۵۵)
#FFFFFF = سفید      (۲۵۵, ۲۵۵, ۲۵۵)
#000000 = سیاه      (۰, ۰, ۰)
#FFA500 = نارنجی     (۲۵۵, ۱۶۵, ۰)
```

**تجزیه #FFA500:**
```
FF = ۲۵۵ (قرمز)
A5 = ۱۶۵ (سبز)
۰۰ = ۰   (آبی)
```

### ۲. آدرس‌های حافظه

وقتی برنامه‌نویسان اشکال‌زدایی می‌کنند، آدرس‌های حافظه را می‌بینند:
```
0x7FFF_A000  (یک آدرس حافظه معمولی)
0x0040_0000  (آدرس دیگر)
```

پیشوند "0x" یعنی "این هگزادسیمال است."

### ۳. آدرس‌های MAC

هر دستگاه شبکه یک آدرس MAC منحصر به فرد دارد:
```
00:1A:2B:3C:4D:5E
```

هر جفت یک بایت هگزادسیمال است (دو رقم هگزادسیمال).

### ۴. کاراکترهای یونیکد

نقاط کد یونیکد به صورت هگزادسیمال نوشته می‌شوند:
```
U+0041 = 'A'
U+0627 = حرف عربی 'ا'
U+1F600 = ایموجی '😀'
```

### ۵. کدهای خطا

صفحه آبی ویندوز، خطاهای سیستم:
```
Error 0x80070057 (هگزادسیمال)
= Error 2147942487 (دهدهی)
```

نوشتن در هگزادسیمال خیلی کوتاه‌تر است!

## نمادگذاری هگزادسیمال

### روش‌های مختلف نوشتن "این هگزادسیمال است"

| نمادگذاری | مثال | استفاده در |
|-----------|------|-----------|
| پیشوند 0x | 0xFF | C، C++، Java، Python |
| پیشوند # | #FF5733 | رنگ‌های وب، CSS |
| پیشوند 0h | 0hFF | برخی اسمبلی |
| پیشوند $ | $FF | برخی اسمبلی، Pascal |
| پیشوند &H | &HFF | Visual Basic |
| پیشوند % | %11111111 | دودویی در برخی زمینه‌ها |

**مهم:** در برنامه‌نویسی، `0xFF` یعنی "عدد هگزادسیمال FF،" نه "۰ ضربدر FF!"

## نکات کلیدی

۱. **هگزادسیمال مبنای ۱۶ است**: از ۰-۹ و A-F استفاده می‌کند
۲. **تطابق کامل با دودویی**: ۴ بیت = ۱ رقم هگزادسیمال
۳. **تبدیل آسان است**: گروه‌بندی دودویی در ۴ تایی برای هگزادسیمال
۴. **رایج در محاسبات**: رنگ‌ها، آدرس‌های حافظه، کدهای خطا
۵. **پیشوند 0x**: روش استاندارد برای نشان دادن هگزادسیمال در برنامه‌نویسی

- تمرین تبدیل بین هر سه سیستم (دهدهی، دودویی، هگزادسیمال)
