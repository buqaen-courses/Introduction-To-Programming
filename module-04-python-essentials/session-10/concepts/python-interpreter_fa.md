# مفسر پایتون: مترجم کد شما

## آنچه خواهید آموخت
- مفسر پایتون چه کاری انجام می‌دهد
- چگونه کد پایتون را به دو روش مختلف اجرا کنیم
- چرا پایتون یک "زبان مفسری" نامیده می‌شود

---

## مفسر پایتون چیست؟

مفسر پایتون را به عنوان یک **مترجم** که بین شما و کامپیوتر می‌نشیند تصور کنید.

```
شما (کد پایتون می‌نویسید)
    ↓
مفسر پایتون (ترجمه می‌کند)
    ↓
کامپیوتر (اجرا می‌کند)
```

مفسر کد پایتون شما را خط به خط می‌خواند و به کامپیوتر می‌گوید چه کاری انجام دهد. برخلاف برخی زبان‌های برنامه‌نویسی دیگر که نیاز به "کامپایل" (ترجمه همه‌چیز یک‌بار قبل از اجرا) دارند، پایتون کد شما را یک خط در میان ترجمه و اجرا می‌کند.

### تشبیه: مفسر مثل یک سرآشپز است که دستورالعمل می‌خواند

تصور کنید در حال آشپزی با یک سرآشپز هستید:
- **زبان‌های کامپایل‌شده:** کل دستورالعمل را می‌نویسید، سرآشپز آن را حفظ می‌کند، سپس همه‌چیز را می‌پزد
- **پایتون (مفسی):** هر مرحله را بلند می‌خوانید، و سرآشپز فوراً آن را انجام می‌دهد

این پایتون را برای یادگیری عالی می‌کند زیرا بازخورد فوری می‌گیرید!

---

## دو روش برای اجرای کد پایتون

### روش ۱: حالت تعاملی (REPL)

REPL مخفف **R**ead, **E**valuate, **P**rint, **L**oop است. مثل داشتن یک مکالمه با پایتون است.

#### نحوه شروع حالت تعاملی

ترمینال/خط فرمان خود را باز کنید و تایپ کنید:

```bash
python
```

یا در برخی سیستم‌ها (به‌ویژه macOS/Linux):

```bash
python3
```

چیزی شبیه به این را خواهید دید:

```
Python 3.9.7 (default, Sep 16 2021, 13:09:58)
[GCC 7.5.0] :: Anaconda, Inc. on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>
```

`>>>` نشانه **prompt** نامیده می‌شود - یعنی پایتون می‌گوید "آماده‌ام، چیزی تایپ کن!"

#### خودتان امتحان کنید!

این خطوط را یکی یکی بعد از prompt `>>>` تایپ کنید:

```python
>>> print("Hello, World!")
Hello, World!

>>> 5 + 3
8

>>> name = "Alice"
>>> print(name)
Alice
```

توجه کنید که پایتون فوراً بعد از هر خط نتیجه را به شما نشان می‌دهد!

#### چه زمانی از حالت تعاملی استفاده کنیم

✅ **عالی برای:**
- آزمایش قطعات کوچک کد
- یادگیری و آزمایش
- محاسبات سریع
- بررسی اینکه آیا چیزی کار می‌کند

❌ **مناسب نیست برای:**
- برنامه‌هایی که می‌خواهید ذخیره کنید
- پروژه‌های بزرگ
- کدی که می‌خواهید به اشتراک بگذارید

---

### روش ۲: حالت اسکریپت (نوشتن فایل‌ها)

حالت اسکریپت یعنی نوشتن کد پایتون خود در یک فایل و اجرای همه آن یک‌باره.

#### مرحله ۱: ایجاد یک فایل پایتون

یک فایل به نام `hello.py` با این محتوا بسازید:

```python
# اولین برنامه پایتون من!
print("Hello, World!")
name = input("What's your name? ")
print(f"Nice to meet you, {name}!")
```

**مهم:** فایل‌های پایتون همیشه با `.py` تمام می‌شوند

#### مرحله ۲: اجرای فایل

ترمینال را در همان پوشه‌ای که فایل شماست باز کنید و تایپ کنید:

```bash
python hello.py
```

یا در برخی سیستم‌ها:

```bash
python3 hello.py
```

خواهید دید:

```
Hello, World!
What's your name? Alice
Nice to meet you, Alice!
```

#### چه زمانی از حالت اسکریپت استفاده کنیم

✅ **عالی برای:**
- برنامه‌هایی که می‌خواهید ذخیره و دوباره اجرا کنید
- به اشتراک‌گذاری کد با دیگران
- ساخت برنامه‌های کامل
- نوشتن برنامه‌های طولانی‌تر

---

## چه اتفاقی می‌افتد وقتی پایتون را اجرا می‌کنید؟

در اینجا نسخه ساده آنچه در پشت صحنه اتفاق می‌افتد آمده است:

```
فایل کد شما (.py)
    ↓
پایتون فایل را می‌خواند
    ↓
پایتون به بایت‌کد تبدیل می‌کند (فرمت دوست‌دار کامپیوتر)
    ↓
ماشین مجازی پایتون (PVM) اجرا می‌کند
    ↓
نتایج را می‌بینید!
```

نگران جزئیات فنی نباشید - پایتون همه اینها را به صورت خودکار انجام می‌دهد!

## نسخه‌های مختلف پایتون

ممکن است درباره پایتون ۲ و پایتون ۳ بشنوید. در اینجا آنچه باید بدانید آمده است:

| نسخه | وضعیت | چه چیزی استفاده کنیم |
|---------|--------|-------------|
| Python ۲ | قدیمی، دیگر پشتیبانی نمی‌شود | ❌ استفاده نکن |
| Python ۳ | فعلی، در حال توسعه فعال | ✅ از این استفاده کن! |

**همیشه از پایتون ۳** برای هر پروژه یا یادگیری جدید استفاده کنید.

### بررسی نسخه پایتون شما

در حالت تعاملی، نسخه در بالا نشان داده می‌شود. یا می‌توانید اجرا کنید:

```bash
python --version
# یا
python3 --version
```

## اشتباهات رایج مبتدیان

### اشتباه ۱: فراموش کردن ذخیره قبل از اجرا

اگر تغییراتی در فایل `.py` خود ایجاد می‌کنید، باید قبل از اجرای دوباره آن را **ذخیره** کنید. پایتون آنچه روی دیسک ذخیره شده را اجرا می‌کند، نه آنچه در ویرایشگر شماست.

### اشتباه ۲: پسوند فایل اشتباه

اطمینان حاصل کنید که فایل‌های شما با `.py` تمام می‌شوند:
- ✅ `myprogram.py` (صحیح)
- ❌ `myprogram.txt` (اشتباه)
- ❌ `myprogram` (اشتباه - بدون پسوند)

### اشتباه ۳: تلاش برای اجرای کد با `>>>` در یک فایل

هنگام نوشتن یک فایل `.py`، prompt `>>>` را وارد نکنید:

```python
# ❌ اشتباه - این را انجام نده
>>> print("Hello")

# ✅ صحیح - این را انجام بده
print("Hello")
```

### اشتباه ۴: پوشه اشتباه

اطمینان حاصل کنید که در همان پوشه‌ای که فایل پایتون شماست هنگام اجرا قرار دارید:

```bash
# بررسی کنید در کدام پوشه هستید
pwd  # در macOS/Linux
cd   # در ویندوز

# لیست فایل‌ها را برای دیدن اینکه آیا فایل پایتون شما آنجاست ببینید
ls        # در macOS/Linux
dir       # در ویندوز
```

## خودتان امتحان کنید: تمرین‌ها

### تمرین ۱: تمرین حالت تعاملی

۱. با تایپ `python` یا `python3` حالت تعاملی را باز کنید
۲. این محاسبات را امتحان کنید:
   - `10 * 5`
   - `100 / 4`
   - `2 ** 8` (یعنی ۲ به توان ۸)
۳. یک متغیر بسازید: `favorite_number = 7`
۴. آن را چاپ کنید: `print(favorite_number)`

### تمرین ۲: اولین اسکریپت شما

۱. یک فایل به نام `first_program.py` بسازید
۲. این کد را بنویسید:
   ```python
   print("My First Python Program")
   print("I am learning Python!")
   ```
۳. فایل را ذخیره کنید
۴. از ترمینال اجرا کنید: `python first_program.py`

### تمرین ۳: اسکریپت سلام تعاملی

۱. یک فایل به نام `greeting.py` بسازید
۲. این کد را بنویسید:
   ```python
   name = input("Enter your name: ")
   age = input("Enter your age: ")
   print(f"Hello {name}! You are {age} years old.")
   ```
۳. ذخیره و اجرا کنید
