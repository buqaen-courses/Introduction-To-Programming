## مقدمه: آغاز، نه پایان

این دوره به شما یک پایه داد. اکنون زمان آن رسیده که بر آن بنا کنید. این راهنما به شما کمک می‌کند تصمیم بگیرید چه چیزی را بعداً یاد بگیرید و چگونه به عنوان یک برنامه‌نویس به رشد خود ادامه دهید.

## بخش ۴: ابزارهای ضروری برای یادگیری

### کنترل نسخه: Git

**چرا**: ردیابی تغییرات، همکاری، پشتیبان کد

**یاد بگیرید**:
```bash
git init          # شروع یک مخزن
git add .         # آماده‌سازی تغییرات
git commit -m ""  # ذخیره تغییرات
git status        # دیدن وضعیت
git log           # مشاهده تاریخچه
git push          # آپلود به GitHub
git pull          # دانلود از GitHub
```

**منابع**:
- GitHub Learning Lab (lab.github.com)
- کتاب «Pro Git» (آنلاین رایگان)

### ویرایشگر کد: VS Code

**چرا**: حرفه‌ای، رایگان، ویژگی‌های زیاد

**یاد بگیرید**:
- افزونه‌ها (Python، Git، تم‌ها)
- دیباگ کردن
- ترمینال یکپارچه
- میانبرهای صفحه‌کلید

**منابع**:
- مستندات VS Code
- آموزش‌های YouTube

### محیط‌های مجازی

**چرا**: مدیریت وابستگی‌های پروژه

**یاد بگیرید**:
```bash
python -m venv myenv      # ایجاد محیط
myenv\Scripts\activate    # فعال‌سازی (ویندوز)
source myenv/bin/activate # فعال‌سازی (مک/لینوکس)
pip install package       # نصب بسته‌ها
```

### مدیر بسته: pip

**یاد بگیرید**:
```bash
pip install package       # نصب یک بسته
pip install -r requirements.txt  # نصب از لیست
pip list                  # دیدن بسته‌های نصب شده
pip freeze                # تولید requirements
```

## بخش ۹: موانع رایج که باید اجتناب کنید

### ۱. سندروم شیء براق

**مشکل**: مرتباً تعویض به آموزش‌ها/دوره‌های جدید

**راه‌حل**: قبل از حرکت، آنچه را شروع کرده‌اید تمام کنید

### ۲. جهنم آموزش

**مشکل**: فقط دنبال کردن آموزش‌ها، هرگز ساختن مستقل

**راه‌حل**: در ۸۰٪ درک متوقف شوید، شروع به ساختن کنید

### ۳. تله مقایسه

**مشکل**: «دیگران خیلی جلوتر هستند»

**راه‌حل**: خود را با گذشته خود مقایسه کنید، نه با دیگران

### ۴. کمال‌گرایی

**مشکل**: هرگز تمام نکردن چون کامل نیست

**راه‌حل**: ارسال کنید، سپس بهبود دهید. انجام شده > کامل.

### ۵. یادگیری بدون انجام

**مشکل**: خواندن/تماشا بدون تمرین

**راه‌حل**: همراه کد بنویسید، سپس نسخه خود را بسازید

شما اولین گام را در یک سفر هیجان‌انگیز برداشته‌اید. برنامه‌نویسی درهایی را به سوی موارد زیر باز می‌کند:
- بیان خلاقانه
- حل مسئله
- فرصت‌های شغلی
- یادگیری مستمر
- ساختن چیزهایی که اهمیت دارند

**به یاد داشته باشید**:
- همه به عنوان یک مبتدی شروع کردند
- اشتباهات بخشی از یادگیری هستند
- تداوم بر شدت غلبه می‌کند
